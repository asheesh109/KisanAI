# Contributor Journey

This document provides a concise onboarding checklist and the expected journey for new contributors.

## Onboarding Checklist

1. Fork the repository on GitHub
2. Clone your fork locally
3. Add upstream remote: `git remote add upstream https://github.com/asheesh109/KisanAI.git`
4. Create a branch following conventions: `feat/your-feature`
5. Claim an issue and post a short plan
6. Implement changes and add tests/docs
7. Run lint and tests locally
8. Open a PR and request review

## Contributor Flow (High level)

1. Discover an issue (labels: `good first issue`, `beginner`)
2. Claim it and create a branch
3. Implement & test changes
4. Open PR and iterate on feedback
5. Merge and get credited in `README.md`/contributors

## How to Choose Issues

- Start with `good first issue` to learn the codebase.
- Pick issues that have clear acceptance criteria and limited scope.
- Prefer documentation, tests, or small UI changes for first contributions.

## Asking for Help

- Use GitHub Discussions for design or process questions.
- Comment on the issue with blockers and steps you've tried.
- Tag `@asheesh109` for maintainers' attention when needed.

## How Reviews Work

- Reviews typically happen within 24-72 hours.
- Address comments by pushing to the same branch; request re-review.
- Maintainers may squash commits during merge; keep commit messages descriptive.
