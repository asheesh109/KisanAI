# Hacktoberfest Contribution Guide

This project welcomes Hacktoberfest contributions. Follow these guidelines to ensure your PRs count and are helpful.

## Contribution Expectations

- Focus on meaningful improvements: documentation, bug fixes, tests, and small features.
- Avoid trivial or low-value changes (spam PRs will be closed).

## Valid PR Rules

- PRs should be linked to an issue or clearly describe the problem they solve.
- PRs must follow the project's commit and branch conventions.
- Each PR should include tests or documentation updates when applicable.

## Spam PR Warning

- Do not submit mass-formatting-only PRs, typo-only PRs across many files, or empty changes.
- Maintainers will close PRs that do not add clear value.

## Quality Contribution Guide

1. Search `ISSUE_BANK.md` for `good first issue` or `beginner` tasks.
2. Claim the issue and ask for clarification if necessary.
3. Submit a focused PR with tests and documentation updates.
