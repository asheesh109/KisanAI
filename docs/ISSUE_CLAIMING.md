# Issue Claiming Guide

When you want to work on an issue:

1. Comment on the issue: "I would like to work on this — assigning to myself."
2. Wait for a maintainer to acknowledge and assign (within 48 hours).
3. Create a branch named `issue/<issue-number>-short-desc`.
4. Post updates regularly (e.g., progress, blockers) as comments on the issue.
5. When ready, open a PR referencing the issue and include test instructions.

If you're stalled for >7 days, leave a comment so maintainers can reassign or help.
