# GSSoC Contributor Guide

Welcome GSSoC applicants and mentors! This guide helps aspiring contributors onboard and contribute effectively during the GSSoC program.

## Onboarding

1. Read `CONTRIBUTING.md` and `CODE_OF_CONDUCT.md`.
2. Introduce yourself in GitHub Discussions and mention your interest in GSSoC.
3. Look for `good first issue` and `beginner` labels in `ISSUE_BANK.md`.

## How to Start Contributing

- Choose 2-3 beginner-friendly issues and comment to claim them.
- Ask for a mentor by tagging `@asheesh109` and adding `gssoc-mentor-request` label.
- Submit small PRs early to get feedback and iterate.

## Beginner-Friendly Guide

- Start with documentation, tests, or UI tweaks labeled `good first issue`.
- Follow branch and commit conventions in `CONTRIBUTING.md`.
- Keep PRs small and include test instructions.

## Issue Difficulty Explanation

- **Good First**: 30–120 minutes; minimal code changes; safe to attempt without deep context.
- **Beginner**: 2–6 hours; requires basic understanding of code structure.
- **Intermediate**: 6–16 hours; needs familiarity with project services.
- **Advanced**: 16+ hours; architecture-level work.

## Mentorship & Support

- Mentors: `@asheesh109` (primary maintainer) and assigned mentors.
- To request mentorship: comment on the issue, add `gssoc-mentor-request`, and provide your availability.
- Mentors will provide a plan, review PRs, and help unblock contributors.

## Evaluation Criteria

- PR quality and adherence to guidelines
- Timely responses to requested changes
- Tests and documentation accompanying code
