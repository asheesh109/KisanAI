## 🚀 How to Use

### 1. Enable PWA in Production

```bash
npm run build
npm start
# PWA automatically enabled in production
```

### 2. Test Offline in Development

```bash
npm run dev
# Go to Chrome DevTools → Application → Service Workers
# Check "Offline" to simulate offline mode
```

### 3. Cache Data Manually

```typescript
import { saveData, getData } from '@/lib/indexedDB';

// Save data
await saveData('my_key', data, 24 * 60 * 60 * 1000); // 24 hour TTL

// Retrieve data
const cached = await getData('my_key');
```

### 4. Detect Offline Status

```typescript
import { useOfflineStatus } from '@/hooks/useOfflineStatus';

export default function MyComponent() {
  const { isOffline } = useOfflineStatus();
  
  return isOffline ? <OfflineView /> : <OnlineView />;
}
```

## 📋 Next Steps: Integration Guide

### For Each Feature Page Add:

#### 1. Import statements
```typescript
import { saveData, getData, getTimeAgo } from '@/lib/indexedDB';
import { useOfflineStatus } from '@/hooks/useOfflineStatus';
```

#### 2. Define cache key
```typescript
const CACHE_KEY = 'feature_name_data';
const CACHE_TTL = 24 * 60 * 60 * 1000; // 24 hours
```

#### 3. In useEffect, save data after API call
```typescript
// After successful API response
await saveData(CACHE_KEY, responseData, CACHE_TTL);
```

#### 4. On error, fallback to cache
```typescript
catch (error) {
  const cached = await getData(CACHE_KEY);
  if (cached) setData(cached);
}
```

#### 5. When offline, load cache automatically
```typescript
useEffect(() => {
  if (isOffline) {
    getData(CACHE_KEY).then(setData);
  }
}, [isOffline]);
```

#### 6. Show cache indicator
```typescript
{fromCache && (
  <small className="text-orange-600">
    📦 Cached {getTimeAgo(timestamp)}
  </small>
)}
```

## 🔧 Configuration Summary

### next-pwa Settings

| Setting | Value | Purpose |
|---------|-------|---------|
| `dest` | 'public' | Output directory |
| `register` | true | Auto-register SW |
| `skipWaiting` | true | Immediate activation |
| `disable` | dev mode | Only in production |

### Runtime Caching

| Pattern | Strategy | TTL | Items |
|---------|----------|-----|-------|
| API calls | NetworkFirst | 24h | JSON responses |
| Static assets | CacheFirst | 30d | CSS, JS, images |
| Google Fonts | CacheFirst | 1y | Font files |
| Translations | CacheFirst | 7d | Translation API |

## 🐛 Debugging Tips

### Service Worker Issues

1. **SW not registering:**
   ```bash
   # Check DevTools console for SW errors
   # Ensure HTTPS or localhost
   # Verify manifest.json is valid
   ```

2. **Cache not updating:**
   ```javascript
   // In browser console
   navigator.serviceWorker.getRegistrations().then(regs => {
     regs.forEach(reg => reg.unregister());
   });
   // Hard refresh with Ctrl+Shift+R
   ```

3. **IndexedDB errors:**
   ```javascript
   // Check available quota
   navigator.storage.estimate().then(({usage, quota}) => {
     console.log(`Used: ${usage}B of ${quota}B`);
   });
   ```

### Enable Logging

```javascript
// In browser console
localStorage.setItem('debug', '*');
// Reload page, check logs
```

## 🔒 Security Notes

1. **HTTPS Required**
   - PWA only works on HTTPS (or localhost)
   - Ensure all APIs use HTTPS

2. **Safe Data Caching**
   - ✅ Safe to cache: Schemes, prices, weather, translations
   - ❌ Never cache: Passwords, tokens, sensitive user data

3. **Cache Invalidation**
   - Use TTL (time-to-live) for data freshness
   - Implement version checks for API changes
   - Clear cache on security events

## 🎯 Migration Tips

### For Existing Features

1. **Identify cacheable data**
   - Schemes, prices, weather, user preferences

2. **Choose appropriate TTL**
   - Schemes: 24 hours (changes infrequently)
   - Prices: 4-6 hours (updated daily)
   - Weather: 12 hours (forecasts update daily)
   - User settings: 7 days (rarely change)

3. **Add cache save logic**
   - After successful API response
   - With proper error handling
   - Update UI to show cached status

4. **Test thoroughly**
   - Online and offline scenarios
   - Network failures
   - Cache expiration
   - Data consistency
